# Issue Tracking

## System Of Record

- For the product itself, repository history and harness planning artifacts are the initial system of record.
- For observed feature state, the source of truth is the work-item directory under `docs/exec-plans/current/`.

## Intake Workflow

- Capture planned work as harness work items.
- Implement features against those artifacts.
- Use the TUI to observe the live state of the work item as the docs evolve during development.
